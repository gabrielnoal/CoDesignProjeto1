  3. Estação Meteorológica

      Nome: Estação Meteorológica
      Curso: Instrumentação e Medição
      Periodo: Primeiro
      Design: Sim
      Descrição: Descricao: construcao de uma estacao meteorologica em uma placa de circuito impresso. A estacao continha 4 sensores (temperatura, pressao, luminosidade e umidade), alem de uma tela de lcd. O projeto se iniciou em abril e se estendeu ate o dia 15 de junho de 2016. Nessa projeto me encarreguei de fazer as medicoes para a calibracao do LM35 e do DHT22.

    • O projeto envolveu software? Sim, para este projeto foram usados o Excel, o software do Arduino e o Fritzing para a confecção da PCB. O grupo usou o Excel para armazenar os dados das medições com cada sensor, gerar gráficos e para calcular as incertezas que foram incluídas no código de Arduino posteriormente para calibrar os sensores. O software do Arduino foi utilizado para escrever os códigos de cada sensor que foram posteriormente compilados no microcontrolador.
    • O projeto envolveu fabricação? Sim, para esse projeto utilizamos a fresadora do FabLab para a confecção da placa de circuito impresso (PCB) e também uma solda para soldar os componentes (resistores, sensores e LCD) na placa.
    • O projeto envolveu eletrônica? Sim, esse projeto é de modo geral um projeto de eletrônica. Utilizamos o Arduino Uno e diversos sensores analógicos e digitais, além de uma tela de LCD para mostrar em seu display as medições de cada sensor (temperatura, pressão, umidade relativa do ar e luminosidade). Dessa forma, pode-se concluir que bastante eletrônica foi empregada nesse projeto do primeiro semestre.
    • O projeto envolveu design centrado no usuário? De certo forma sim, pois poderíamos ter deixado a estação em uma protoboard, tornando-a mais difícil de ser manuseada e possível de se desconectar algum jumper, por exemplo. Assim, foi confeccionada uma placa para que fosse possível soldar todos os componentes sem o uso de jumpers e dessa forma fica mais simples para o usuário.
    • O projeto envolveu escrita técnica e científica? Sim, já que um relatório final foi cobrado. Nesse relatório precisamos colocar o passo a passo explicado de cada etapa do projeto (medição, fabricação, calibração) e para isso foi necessário o uso de uma linguagem cientifica, já que o projeto englobou conhecimentos de Estatística, Química, Eletrônica entre outros.
    • Qual foi o curso em que o projeto foi feito? Instrumentação e Medição.
    • Qual foi a data em que o projeto foi feito? Começo de Abril de 2016 até meio de Junho 2016.
    • Qual foi o período em que o projeto foi feito? O projeto foi feito ao longo do primeiro semestre da faculdade.
    • Qual foi a duração do projeto? Aproximadamente 2 meses.
    • Qual foi o papel do aluno no projeto? Nesse projeto me encarreguei de fazer as medições com os sensores que precisaram ser calibrados (LM35 e DHT22). Além disso, me encarreguei de escrever parte dos textos do relatório e ajudei na parte de fabricação com a solda e o software de confecção da placa.

  4. Capotruz

      Nome: Capotruz
      Curso: Natureza do Design
      Periodo: Primeiro
      Design: Sim
      Descrição:Descrição: manufatura de um brinquedo infantil que tenha como público alvo crianças de 7 a 9 anos de idade. O brinquedo era composto de canos pvc, uma válvula, tinta spray e ovos de isopor. Consiste em uma arma de gás comprimido que era preenchida com ar advindo de uma bomba de bicicleta. O projeto se iniciou em abril e se estendeu ate a metade de junho. Nesse projeto tive o papel da junção dos canos pvc e a pintura do brinquedo.

    • O projeto envolveu software? Não, nesse projeto o trabalho manual prevaleceu. A única vez que algum tipo de software foi utilizado foi para atualizar a wiki do projeto ou para tirar dúvidas pontuais. O FabLab , laboratório da faculdade, dispunha de alguns materiais a serem utilizados como colas e serras.
    • O projeto envolveu fabricação? Sim, como dito anteriormente, a fabricação tomou conta da maior parte do projeto. Por se usarem muitas peças, colas e serras foram definitivamente as ferramentas mais utilizadas. Sprays também foram utilizados para o design do brinquedo.
    • O projeto envolveu eletrônica? Não, não houve envolvimento com eletrônica. O projeto consistiu em uma arma de gás comprimido que simulava um avestruz que cospia ovos de isopor. Dessa forma, a arma era composta de canos PVC, uma válvula, ovos, pintura e uma bomba de bicicleta. Com isso não houve parte eletrônica do trabalho, apenas materiais.
    • O projeto envolveu design centrado no usuário? Com certeza, já que o público alvo já era sabido ao começo do projeto. O grupo fez entrevistas com crianças na faixa de 7 a 9 anos de idade para saber sua opinião à respeito do que seria um brinquedo legal. Depois de colhidos os depoimentos, a ideia da arma parecia cada vez mais interessante e o grupo decidiu por concretizá-la.
    • O projeto envolveu escrita técnica e científica? Não, já que o a única forma de documentação era a página da wiki do projeto em que a escrita não possuía rigor técnico e sim um texto informativo sobre do que se tratava o projeto.
    • Qual foi o curso em que o projeto foi feito? Natureza do Design.
    • Qual foi a data em que o projeto foi feito? Começo de abril até a metade de junho.
    • Qual foi o período em que o projeto foi feito? O projeto foi feito ao longo do primeiro semestre da faculdade.
    • Qual foi a duração do projeto? Cerca de 2 meses.
    • Qual foi o papel do aluno no projeto? Fui responsável pela junção das peças da arma de ar comprimido e a pintura do exterior.

Gabriel Noal, Eduardo Gebara, Breno Nakad
